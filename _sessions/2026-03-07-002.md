## Decisions Made
- **Topic-first architecture**: Policymakers browse by topic (like Exemplars), not by individual paper
- **Preact over React**: 3KB vs 45KB -- better for Nepal bandwidth constraints
- **No runtime AI**: Everything static, zero API costs. Build a Brief assembles existing paper data, no AI synthesis
- **Verification workflow**: Papers have status field (auto-processed/verified). Auto-processed shows warning banner
- **PDF via browser print**: Custom print CSS with forced colors, branded header/footer. No server-side PDF generation needed
- **Print CSS approach**: Explicit class names (print-badge, relevance-dot-filled) instead of relying on Tailwind's generated class selectors, because Tailwind v4 uses CSS custom properties that don't survive print stylesheets
- **Paper 1-pager structure**: Changed `<header>` to `<div>` for bibliographic info to avoid being hidden by print CSS `header { display: none }`
- **Build a Brief**: Static assembly from existing data (topic + country filters) rather than AI-generated synthesis -- achieves the goal without funding
